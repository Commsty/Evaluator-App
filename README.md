# Calculator Backend

REST API для калькулятора с поддержкой математических выражений.

## Актуальные даты для проекта
Июль - Август 2025

## Описание проекта

Calculator Backend - это веб-сервис на Go, который предоставляет REST API для выполнения математических вычислений и хранения результатов в базе данных. Проект демонстрирует чистую архитектуру и лучшие практики разработки на Go.

## Возможности

- **Вычисление выражений** - поддержка базовых математических операций
- **Хранение результатов** - сохранение всех вычислений в PostgreSQL
- **CRUD операции** - полное управление вычислениями
- **Валидация** - проверка входных данных
- **REST API** - стандартный HTTP интерфейс

## Технологии

- **Язык**: Go 1.24.4
- **Веб-фреймворк**: Echo v4
- **База данных**: PostgreSQL
- **ORM**: GORM
- **Вычисления**: govaluate
- **Идентификаторы**: UUID

## API Эндпоинты

### Получить все вычисления
```http
GET /calculations
```

### Создать вычисление
```http
POST /calculations
Content-Type: application/json

{
  "expression": "2+2*3"
}
```

### Обновить вычисление
```http
PATCH /calculations/{id}
Content-Type: application/json

{
  "expression": "10+5"
}
```

### Удалить вычисление
```http
DELETE /calculations/{id}
```

## Установка и запуск

### 1. Клонирование и установка зависимостей

```bash
git clone <repository-url>
cd CalculatorAppBackend
go mod download
```

### 2. Настройка базы данных

Убедитесь что PostgreSQL запущен и доступен. Измените параметры подключения в коде при необходимости.

### 3. Запуск

```bash
# Разработка
go run cmd/main.go

# Сборка
go build -o bin/calculator cmd/main.go
./bin/calculator
```

Сервис будет доступен по адресу: `http://localhost:8080`

## Примеры использования

### Создание вычисления
```bash
curl -X POST http://localhost:8080/calculations \
  -H "Content-Type: application/json" \
  -d '{"expression": "15+25*2"}'

# Ответ:
{
  "id": "uuid-string",
  "expression": "15+25*2",
  "result": "65"
}
```

### Получение всех вычислений
```bash
curl http://localhost:8080/calculations

# Ответ:
[
  {
    "id": "uuid-string",
    "expression": "15+25*2",
    "result": "65"
  }
]
```

## Валидация

API включает валидацию:

- **Expression** не может быть пустым
- **ID** должен быть валидным UUID
- **Математические выражения** проверяются на корректность

## Обработка ошибок

API возвращает стандартные HTTP статусы:
- `200` - Успешное выполнение
- `201` - Успешное создание
- `204` - Успешное удаление
- `400` - Ошибка валидации или запроса
- `500` - Внутренняя ошибка сервера
